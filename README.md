## Drupal Installation and Development Guide

This section provides comprehensive instructions for installing, upgrading, and developing Drupal websites using Composer, Docker, and DDEV.

### Installing Drupal Core with Composer

Drupal Core can be easily installed using Composer, which manages all dependencies automatically.

#### Create a New Drupal Project

```bash
# Install the latest stable Drupal 11
composer create-project drupal/recommended-project my-drupal-site

# Or install Drupal 10 (if you need compatibility with older modules)
composer create-project drupal/recommended-project:^10 my-drupal-site-10

# Navigate to the project directory
cd my-drupal-site
```

#### Manual Installation Steps

If you prefer more control over the installation process:

```bash
# Create project directory
mkdir my-drupal-site
cd my-drupal-site

# Initialize Composer
composer init

# Add Drupal core dependencies
composer require drupal/core:~11.0 --no-interaction

# Install Drupal using the web installer or Drush
./vendor/bin/drush site-install minimal --db-url=sqlite://sites/default/files/.sqlite --site-name=MySite
```

### Upgrading Drupal Core with Composer

Keeping Drupal core up-to-date is essential for security and functionality.

#### Check Current Version

```bash
# Check installed Drupal version
drush status | grep "Drupal version"

# Or via Composer
composer show drupal/core | grep "^name\|^version"
```

#### Upgrade to Latest Minor Version

```bash
# Backup your database and files first!
drush sql-dump > backup-$(date +%Y%m%d).sql

# Update Drupal core to the latest minor version
composer update drupal/core --with-dependencies

# Run database updates
drush updatedb

# Clear caches
drush cr

# Verify the upgrade
drush status
```

#### Upgrade to Major Version (e.g., Drupal 10 to 11)

```bash
# IMPORTANT: Always check module compatibility before upgrading!
# Visit https://www.drupal.org/project/drupal/releases/11.0.0
# Verify all contrib modules have Drupal 11 versions

# Backup everything!
drush sql-dump > backup-$(date +%Y%m%d).sql
tar -czf files-backup-$(date +%Y%m%d).tar.gz web/sites/default/files

# Update composer.json for Drupal 11
composer require drupal/core:~11.0 --no-update

# Update all dependencies
composer update

# Run database updates
drush updatedb

# Clear all caches
drush cr

# Verify the upgrade
drush status
```

#### Automated Security Updates

Enable automatic security updates for Drupal core:

```bash
# Install the automatic_updates module
composer require drupal/automatic_updates

# Enable the module
drush en automatic_updates -y

# Configure via admin interface or Drush
drush automatic_updates:settings
```

### Using Docker for Drupal Development

Docker provides containerized development environments for Drupal.

#### Basic Docker Setup

```bash
# Create docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  drupal:
    image: drupal:11-apache
    ports:
      - "8080:80"
    volumes:
      - ./web:/var/www/html
    environment:
      - DRUPAL_SITE_NAME=MyDrupalSite
      - DRUPAL_USER=admin
      - DRUPAL_PASS=secure_password
    depends_on:
      - db

  db:
    image: mysql:8.0
    environment:
      - MYSQL_DATABASE=drupal
      - MYSQL_USER=drupal
      - MYSQL_PASSWORD=drupal
      - MYSQL_ROOT_PASSWORD=root_password
    volumes:
      - mysql_data:/var/lib/mysql

volumes:
  mysql_data:
EOF

# Start containers
docker-compose up -d

# Access Drupal at http://localhost:8080
```

#### Docker Compose for Custom Drupal Setup

```bash
# Create advanced docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  php:
    build:
      context: .
      dockerfile: Dockerfile.php
    ports:
      - "9000:9000"
    volumes:
      - ./:/var/www/html
    environment:
      - PHP_MEMORY_LIMIT=256M
      - PHP_MAX_EXECUTION_TIME=300
    depends_on:
      - db
      - redis

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./:/var/www/html
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - php

  db:
    image: mysql:8.0
    environment:
      - MYSQL_DATABASE=drupal
      - MYSQL_USER=drupal
      - MYSQL_PASSWORD=drupal
      - MYSQL_ROOT_PASSWORD=root_password
    volumes:
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:alpine
    ports:
      - "6379:6379"

volumes:
  mysql_data:
EOF

# Create Dockerfile.php
cat > Dockerfile.php << 'EOF'
FROM drupal:11-php8.3-apache

# Install PHP extensions
RUN apt-get update && apt-get install -y \
    libzip-dev \
    unzip \
    && docker-php-ext-install zip pdo pdo_mysql

# Install Composer
COPY --from=composer:2 /usr/bin/composer /usr/bin/composer

WORKDIR /var/www/html
EOF

# Create nginx.conf
cat > nginx.conf << 'EOF'
events {
    worker_connections 1024;
}

http {
    server {
        listen 80;
        server_name localhost;
        root /var/www/html/web;

        location / {
            try_files $uri /index.php?$query_string;
        }

        location ~ \.php$ {
            fastcgi_pass php:9000;
            fastcgi_index index.php;
            include fastcgi_params;
            fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        }
    }
}
EOF

# Build and start
docker-compose build
docker-compose up -d
```

#### Docker Commands for Drupal

```bash
# Start all services
docker-compose up -d

# View logs
docker-compose logs -f drupal

# Stop all services
docker-compose down

# Stop and remove volumes (WARNING: deletes database)
docker-compose down -v

# Access Drush in container
docker-compose exec drupal drush status

# Run tests
docker-compose exec drupal ./vendor/bin/phpunit

# Backup database
docker-compose exec db mysqldump -u root -proot_password drupal > backup.sql

# Restore database
docker-compose exec -T db mysql -u root -proot_password drupal < backup.sql
```

### Using DDEV for Local Drupal Development

DDEV is a Docker-based tool that simplifies local development for Drupal and other PHP applications.

#### Installing DDEV

```bash
# macOS with Homebrew
brew install drud/ddev/ddev

# Linux
curl -L https://github.com/drud/ddev/releases/download/v1.23/ddev_Linux_arm64.tar.gz | tar -xz && sudo mv ddev /usr/local/bin/

# Windows with Chocolatey
choco install ddev

# Verify installation
ddev --version
```

#### Creating a New Drupal Project with DDEV

```bash
# Create project directory
mkdir my-drupal-project
cd my-drupal-project

# Initialize DDEV with Drupal 11
ddev config --project-type=drupal11 --docroot=web --create-docroot

# Start DDEV environment
ddev start

# Install Drupal using Drush
ddev drush site-install standard \
  --db-url=mysql://db:db@db/drupal \
  --site-name="My DDEV Drupal Site" \
  --account-name=admin \
  --account-pass=admin

# Access your site
ddev launch

# Or access via command line
ddev drush uli
```

```bash
# Add a module using Composer (DDEV manages the path automatically)
```

### Creating a Local Drush Link

Instead of installing Drush globally, you can create a local link to use the version installed in your project's vendor directory. This ensures you're using the correct Drush version for your project.

#### Create a Drush Executable Link

```bash
# Navigate to your Drupal project root
cd /path/to/your/drupal-project

# Create a symbolic link to Drush in your local bin directory
mkdir -p ~/bin
ln -s vendor/drush/drush/drush ~/bin/drush

# Add ~/bin to your PATH if not already added
echo 'export PATH="$HOME/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Verify Drush is accessible
which drush
drush --version
```

#### Alternative: Create a Drush Alias

```bash
# Add this to your ~/.bashrc or ~/.zshrc for quick access
alias drush='vendor/drush/drush/drush'

# Apply the alias
source ~/.bashrc

# Or make it permanent
echo "alias drush='vendor/drush/drush/drush'" >> ~/.bashrc
```

#### Using Drush Launcher (Recommended)

Drush Launcher is a separate tool designed to automatically use the local Drush installation:

```bash
# Install Drush Launcher globally
curl -OL https://github.com/drush-ops/drush-launcher/releases/download/0.10.0/drush.phar
chmod +x drush.phar
sudo mv drush.phar /usr/local/bin/drush

# Now 'drush' will automatically use vendor/drush/drush
cd /path/to/your/drupal-project
drush --version

# Verify it's using local Drush
drush status | grep "Drush script"
```

#### Create a Project-Specific Drush Script

```bash
# Create a wrapper script in your project root
cat > drush << 'EOF'
#!/bin/bash
# This script runs the local Drush installation
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
exec php "$SCRIPT_DIR/vendor/drush/drush/drush" "$@"
EOF

chmod +x drush

# Use it from your project root
./drush --version
```

```bash
which drush

# Verify Drush can connect to your site
drush status

# Test a common command
drush cr  # Clear caches

# Check Drush launcher is working
drush launcher:status
```

#### Troubleshooting Drush Link Issues

```bash
# If drush command is not found
echo $PATH | tr ':' '\n' | grep -q "$HOME/bin" && echo "~/bin in PATH" || echo "Add ~/bin to PATH"

# If symbolic link is broken
ls -la ~/bin/drush
# Recreate if needed
rm ~/bin/drush
ln -s /path/to/project/vendor/drush/drush/drush ~/bin/drush

# Check Drush permissions

# Verify PHP can execute Drush
```

```bash
# Check database status
ddev describe

ddev restart db

ddev start
```

#### File Permission Issues

```bash
# Fix file permissions
ddev ssh
cd web/sites/default
chmod 755 files
chmod 644 settings.php

# For development, you might need

```

```bash

ddev exec "COMPOSER_MEMORY_LIMIT=-1 composer require drupal/module_name"

```
